Never defer an identified fix to a future version unless the user has explicitly asked you to scope work that way. If you identify two valid approaches to a problem during implementation, explicitly ask the user which approach they prefer while giving them your recommendation. Do not write a "NOTE: alternative approach" comment in the code and leave the inferior solution in place. A comment describing a better solution you chose not to implement is a deferred bug. explicitly ask the user which approach they prefer — do not make the decision for them by defaulting to the simpler path.
